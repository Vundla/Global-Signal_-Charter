## Codex Principles

### Inscription Rules

1. **All Charters are living documents**: They evolve as context demands
2. **Transparency is covenant**: Every allocation, decision, and lesson is recorded
3. **Guardians rotate**: Leopard, Lion, Hare take turns watching over Charters
4. **Resilience is the law**: Every Charter must embody fault tolerance
5. **Legacy is inheritance**: Charters are written for generations, not moments

### Update Ceremony

When a Charter is amended:

1. Announce the reason for amendment
2. Consult the Guardian of that Charter
3. Inscribe the date and context of change
4. Archive the previous version
5. Broadcast the update to all nodes

## How to Add a New Charter

```elixir
defmodule GlobalSovereign.Codex do
  @doc """
  Inscribes a new Charter into the Codex.
  
  ## Parameters
  - title: The Charter name
  - guardian: :leopard | :lion | :hare | :triad
  - category: Classification (e.g., "Technical", "Financial")
  - verse: The Charter text
  - principles: List of key principles
  - context: Historical/inspirational context
  """
  def inscribe_charter(params) do
    charter = %{
      id: Nanoid.generate(),
      title: params.title,
      guardian: params.guardian,
      category: params.category,
      verse: params.verse,
      principles: params.principles,
      context: params.context,
      inscribed_at: DateTime.utc_now(),
      version: 1
    }
    
    # Store in database
    Repo.insert!(charter)
    
    # Broadcast to all nodes
    Phoenix.PubSub.broadcast(
      GlobalSovereign.PubSub,
      "codex:charters",
      {:new_charter, charter}
    )
    
    # Archive to IPFS for immutability
    {:ok, cid} = ExIPFS.add(Jason.encode!(charter))
    
    Logger.info("Charter inscribed: #{charter.title} (CID: #{cid})")
    
    {:ok, charter}
  end
end
```
